# FORMAL RISK REGISTER

EML-style symbolic work introduces specific risks that must remain explicit:

- complex-domain internal evaluation
- principal-branch dependence
- ln(0), infinities, signed zeros, and extended-real semantics
- cross-platform floating-point divergence
- exact-value snapping vs unsnapped learned weights
- inability of restricted operator sets to cover every desired algebraic extension
- analog / FPGA dynamic-range instability

```text
sandbox first
→ semantics second
→ determinism audit third
→ production only after proof + benchmark alignment
```
